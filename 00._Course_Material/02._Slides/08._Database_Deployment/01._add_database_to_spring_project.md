# Add the connectivity properties and run the project

In `src/main/resources/application.properties`, add the following properties:

```
spring.datasource.url=jdbc:mysql://localhost:3306/worldcountries
spring.datasource.username=root
spring.datasource.password=<password>
```

**Important**: Make sure to change the username and password to your MySQL credentials and the same with the database name.

**Note**: `3306` is the default port for MySQL.

Now, run the project.

# Use Environment Variables Instead

`application.properties` will be pushed to GitHub and your database credentials will become public. You should never push passwords.

Instead, use [environment variables](https://www.jetbrains.com/help/idea/program-arguments-and-environment-variables.html#environment_variables).

In the `Run` menu select `Edit Configurations...`.

If you don't see the environment variables field then click on `Modify options` and choose `Operating System` > `Environment Variables`.

In the environment variables field add the following variables:

```env
SPRING_DATASOURCE_URL=jdbc:mysql://localhost:3306/worldcountries;
SPRING_DATASOURCE_USERNAME=root;
SPRING_DATASOURCE_PASSWORD=<password>;
```

**Important**: Use semicolons to separate the variables. And use underscores in key names instead of dots.

---

# This works!

This works. Remember to delete credentials from `application.properties`.

We solved the problem of pushing credentials to GitHub.

This setup only works locally in IntelliJ without it adding any secrets to the codebase.

**But** this means that we will need to add these environment variables in:

1. GitHub (so that it can be built) and deployed.
2. Azure (so that it can access the database in production).

# Create a `/models/Country.java` class

```java
public class Country {

    private Long id;
    private String name;

    public Country() {}

    public Country(Long id, String name) {
        this.id = id;
        this.name = name;
    }

    public Long getId() {
        return id;
    }

    public void setId(Long id) {
        this.id = id;
    }

    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }

}
```

---

# Create a `/repositories/CountryRepository.java` class

```java
import org.springframework.stereotype.Repository;
import org.springframework.beans.factory.annotation.Value;
import your.package.name.models.Country;          // CHANGE ME!

import java.sql.*;
import java.util.ArrayList;
import java.util.List;

@Repository
public class CountryRepository {
    @Value("${spring.datasource.url}")
    private String dbUrl;
    @Value("${spring.datasource.username}")
    private String username;
    @Value("${spring.datasource.password}")
    private String password;

    public List<Country> findAll() {
        List<Country> countries = new ArrayList<>();
        String sql = "SELECT * FROM countries";

        try (Connection connection = DriverManager.getConnection(dbUrl, username, password)) {
            Statement statement = connection.createStatement();
            ResultSet resultSet = statement.executeQuery(sql);

            while (resultSet.next()) {
                Country country = new Country(resultSet.getLong("id"), resultSet.getString("name"));
                countries.add(country);
            }

        } catch (SQLException e) {
            e.printStackTrace();
        }

        return countries;
    }
}
```

# Create a `/services/CountryService.java` class

```java
import your.package.name.models.Country;                // CHANGE ME!
import your.package.name.repositories.CountryRepository; // CHANGE ME!
import org.springframework.stereotype.Service;

import java.util.List;

@Service
public class CountryService {

    private final CountryRepository countryRepository;

    public CountryService(CountryRepository countryRepository) {
        this.countryRepository = countryRepository;
    }

    public List<Country> getAllCountries() {
        return countryRepository.findAll();
    }
}
```

---

# Create a `/controllers/CountryController.java` class

```java
import your.package.name.models.Country;                // CHANGE ME!
import your.package.name.services.CountryService;       // CHANGE ME!
import org.springframework.stereotype.Controller;
import org.springframework.web.bind.annotation.RestController;
import org.springframework.web.bind.annotation.GetMapping;

import java.util.List;

@RestController
public class CountryController {

    private final CountryService countryService;

    public CountryController(CountryService countryService) {
        this.countryService = countryService;
    }

    @GetMapping("/countries")
    public List<Country> getAllCountries() {
        return countryService.getAllCountries();
    }

}
```

# Remember to add the credentials to GitHub Secrets

It is in the second PDF and the slide title is: "Define environment variables on GitHub as repository secrets"

The secrets you must define are `PROD_DATABASE_URL`, `PROD_DATABASE_USERNAME` and `PROD_DATABASE_PASSWORD`.

Follow the pdf for how to setup the workflow but here is the snippet to add to it:

```yaml
      - name: Build with Maven
        env:
          SPRING_DATASOURCE_URL: jdbc:mysql://${{secrets.PROD_DATABASE_URL}}/worldcountries
          SPRING_DATASOURCE_USERNAME: ${{secrets.PROD_DATABASE_USERNAME}}
          SPRING_DATASOURCE_PASSWORD: ${{secrets.PROD_DATABASE_PASSWORD}}
        run: mvn clean install
```

Remember to change `/worldcountries` to reflect the name of the database that you have created.

# Setting the environment variables in Azure

It is in the second PDF and the slide title is: "Add environment variables to the web app on Azure Portal"

Follow the pdf for where to set up the variables in Azure but the keys are incorrectly named. 

But use `SPRING_DATASOURCE_URL`, `SPRING_DATASOURCE_USERNAME` and `SPRING_DATASOURCE_PASSWORD` as the keys.

For the `SPRING_DATASOURCE_URL`, do it in the following format:

```plaintext
jdbc:mysql://<PROD_DATABASE_URL>:3306/worldcountries?useSSL=true
```

Change the database name (PROD_DATABASE_URL) according to where it is deployed.
