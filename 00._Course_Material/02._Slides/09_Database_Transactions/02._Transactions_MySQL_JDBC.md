# Locks (Isolation Levels)

Locking can happen at different levels of granularity: row, page, table or even database-level.

We can lock a row in MySQL using `SELECT ... FOR UPDATE` inside a transaction.

When a row is locked, other transactions that try to read or write the same row will be blocked until the lock is released (by committing or rolling back the transaction).


---

# Basic Transaction Commands

```sql
-- Begin Transaction
START TRANSACTION;

-- Lock the seat for update
SELECT status FROM seats WHERE seat_id = 1 FOR UPDATE;

-- Reserve the seat if available
UPDATE seats SET status = 'reserved' WHERE seat_id = 1 AND status = 'available';
SELECT ROW_COUNT(); -- Confirm seat was reserved
-- Row_COUNT() = 1 means success, 0 means seat was already reserved
-- Note: ROW_COUNT() must be called immediately after UPDATE

-- Depending on outcome:
-- If ROW_COUNT() = 1:
COMMIT;

-- Otherwise (if ROW_COUNT() = 0):
-- ROLLBACK;
```

Try to run this in one terminal while running transaction / non-transaction commands in between in another terminal.

# Stored Procedures

We can't have conditional logic in SQL (if ... then... else ...) but it's supported in stored procedures.

Stored procedures are routines stored in the database that can encapsulate repeatable logic (like functions in programming languages).

It's beyond the scope of this course but note that they exist.

# JDBC

Let's see how to use transactions in JDBC.

Reuse the same `seat_reservations` database and `seats` table from previous slides.

Remember to change the connection URL, user and password to your own settings.

**Important:** Don't push the credentials with your code. You should use environment variables instead. For the sake of the example, they will be hardcoded here.

```java
import java.sql.Connection;
import java.sql.DriverManager;
import java.sql.PreparedStatement;
import java.sql.SQLException;

public class Main {

	private static final String CONNECTION_URL = "jdbc:mysql://localhost:3306/seat_reservations";
	private static final String USER = "root";
	private static final String PASSWORD = "password";


	public static void main(String[] args) {
        // JDBC code here
	}
}
```

---

# Inside of the main method

Run the code and check the status in the database after.

```java
		Connection connection = null;
		try {
			connection = DriverManager.getConnection(CONNECTION_URL, USER, PASSWORD);

			// Start transaction
			connection.setAutoCommit(false);

			// Lock the row
			String lockQuery = "SELECT status FROM seats WHERE seat_id = 1 FOR UPDATE";
			PreparedStatement lockStatement = connection.prepareStatement(lockQuery);
			lockStatement.executeQuery();

			// Execute the update query
			String updateQuery = "UPDATE seats SET status = 'reserved' WHERE seat_id = 1;";
			PreparedStatement updateStatement = connection.prepareStatement(updateQuery);
			updateStatement.executeUpdate();

			// Commit transaction
			connection.commit();

			lockStatement.close();
			updateStatement.close();

		} catch (SQLException exception) {
			System.out.println("Error occurred. Rolling back changes");
			try {
				if (connection != null) {
					connection.rollback();
				}
			} catch (SQLException rollbackEx) {
				rollbackEx.printStackTrace();
			}
			exception.printStackTrace();
		} finally {
			try {
				if (connection != null) {
					connection.close();
				}				
			} catch (SQLException closeException) {
				closeException.printStackTrace();
			}

		}
```

# Note about transactions in Spring

You can add the following annotation to a method to make it transactional.

For instance `REPEATABLE_READ` ensures that if you read a row twice in the same transaction, you will get the same result. This avoids dirty reads and non-repeatable reads.

```java
@Transactional(isolation = Isolation.REPEATABLE_READ)
public void reserveSeat(int seatId) {
    // code here
}
```
